The symbols can be stored in plain text files.

Symbols can be separated by white spaces (blanc, tab or end of line) or by comma or semicolon. Lines starting with # or // are considered comments and ignored.

Here is an example of a symbols list file contents

msft yhoo; GOOG,
IBM
intc
// this is a comment
#this is another comment

TXN

 

Note - presently, the symbols cannot contain whitespace characters (blanc, tab, new line). This will be fixed in a future release when symbols surrounded by quote characters will be accepted.

If the symbols file contains duplicate symbols (i.e. symbols with the exact same spelling including the case), only the first occurrence will be considered and subsequent occurrences will be ignored.
