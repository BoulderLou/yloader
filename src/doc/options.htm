• Data Source - the source for the historical data, currently Yahoo or Google. The Google data source has several limitations:

    • the earliest date acceptable is Jan 02, 1970

    • can only download adjusted data

    • supports only daily and weekly data (no monthly data)

  • Auto start downloading - when checked, downloading will be initiated as soon as the application starts. Useful the application is used as part of an automated data downloading system, run from the command line.

  • Check for updates at startup - enables auto checking for new releases at startup

  • Simultaneous connections - the number of simultaneous connections. Depending on the network bandwidth, choosing multiple connections will usually improve downloading speed.

  • Add symbol to column - enables/disables adding a column to each data file containing the symbol name. The column number can be between 0 (leftmost column) and 6 (rightmost column) and should be entered in the associated edit box.

  • Data file name transformations - options to configure the data file name generation for symbols

    • Prepend/Append : prepends and/or appends the specified string to the file name for each symbol. For example, using the strings "_" and "test" the file name for the symbol msft would be: _msfttest.csv (assuming the extension is chosen to be "csv")

    • Extension - the extension for all data files. Can contain a leading "." or not. The default value is csv.

  • Create subdirectories for data files - this will save data in a hierarchy of directories under the main data files directory. The hierarchy is organized in 2 levels: the first level will be named using the first letter of the symbol, and the second level will contain the second letter of the symbol, or _ if the symbol has only one letter. For example, if the root data directory is c:\data and the symbol is MSFT, the path to the data file will be: c:\data\m\s\msft.csv. Similarly, the path for IBM will be c:\data\i\b\ibm.csv and the path for A will be c:\data\a\_\a.csv.

  • Log

    • Log file - the path and name of the file that will receive information about the session (see the Log file section). If this field is left blanc, no log file will be generated.

    • Append new log - when checked, each session will append log information at the end of the log file. If not checked, the log file will be overwritten each time.

    • Log only error events - will only save only error events to the log file

  • Data formatting

    • Pad date fields with 0s - when checked, will ensure that month and day fields have exactly 2 characters, even for 1 digit months or days, so Jan 05, 2006 will be written as 01/05/2006

    • Sort bars in ascending order - if on, it will force saving bars sorted from the oldest to the most recent. If off, the order will be reversed with the most recent bars at the top of the file. In the latter case the Update option in the Time range group is also disabled.

    • Date format - the date can be written in US (mm/dd/yyyy - 02/25/2005) or European format (dd/mm/yyyy - 25/02/2005). When updating, the data must have been saved using the same date format as the currently used one.

  • Data validation - settings apply only if the option to Validate prices on the main window is selected

    • Handling of invalid data - determines the action taken in case of invalid data for a symbol

      • As warning - will display a warning message showing the first invalid bar, but will continue downloading data for the symbol

      • As error - will display an error message showing the first invalid bar, and will stop downloading the data for that symbol

    • Volume 0 is invalid - if checked, a bar with volume 0 will be considered invalid (if price validation is on). The action (warning or error) will still be determined based on the pervious setting. If unchecked, a bar with volume 0 will be considered valid.
